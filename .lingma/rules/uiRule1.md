严格执行以下规则，禁止 AI 自行推测尺寸或样式：

【尺寸规则】
1. 所有宽度、高度、圆角、边距、内边距必须完全按照 UI 文件中的 tailwind 或 CSS 数值转换为等比例 rpx。
2. UI 中的 px 单位请按比例转换为 rpx（375px = 750rpx 设计宽度规则）。
3. 按钮高度、宽度、圆角、字体大小禁止随意更改，必须和 UI 完全一致。
4. 禁止使用微信默认按钮样式；所有按钮样式必须完全自定义。

【字体与排版】
5. 字体大小必须严格匹配 UI：如 10px → 20rpx，12px → 24rpx，14px → 28rpx。
6. 行高 line-height 必须与 UI 一致，否则视觉会出明显差异。
7. 字体颜色必须完全一致，不得自动调整或替换为默认色。

【颜色与阴影】
8. 背景色、渐变色、透明度、阴影（box-shadow）必须严格对应 UI 原始值。
9. 颜色禁止“相似替换”，必须使用 UI 的精确颜色值。

【控件规范】
10. 输入框、按钮、卡片组件不得使用小程序默认样式，必须自定义样式。
11. 所有圆角（如 rounded-2xl、rounded-3xl）必须等比例转换并完全一致。

【布局还原】
12. 垂直与水平间距必须根据 UI 100% 还原，不能自动缩放或优化。
13. Flex / grid 排版必须与 UI 完全一致。
14. 禁止删除 UI 中微小的间距（如 2px、4px），这些都会影响视觉。

【禁止自动优化】
15. 禁止 AI 做“视觉美化”、“自动调整”、“自动响应式”。
16. 只能忠实翻译 UI → 微信小程序代码，不允许“智能修改”。
